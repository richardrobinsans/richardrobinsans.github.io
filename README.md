# Landing — APK download ({{APP_NAME}})

Лёгкий одностраничный лендинг для мотивации скачать и установить Android-приложение (прямая установка APK).
Сделан под медленный интернет и бюджетные устройства (тест: Samsung Galaxy A22). Mobile-first + адаптив под десктоп.

## Стек и почему так
- **Чистый HTML + CSS, без фреймворков и без сборки.** Открывается напрямую, деплоится за минуту.
- **0 JS-библиотек.** Единственный скрипт — подстановка текущего года (1 строка). FAQ работает на нативном `<details>`, без JS.
- **0 веб-шрифтов** — системный стек (на Android это Roboto). Это экономит 50–200 КБ и убирает «прыжок» текста на слабой сети.
- **0 растровых картинок в коде** — иконки инлайн-SVG, фон на CSS-градиентах. Логотип — векторный SVG.
- **Вес страницы:** ~24 КБ без сжатия → ~6–8 КБ по сети с gzip/brotli (включается на GitHub Pages / любом CDN автоматически).

## Структура
```
landing/
├── index.html               ← лендинг (разметка + стили + 1 скрипт)
├── worker.js                ← Cloudflare Worker: гео + трекинг + редирект /go
├── apps-script.gs           ← Google Apps Script: приём событий в таблицу
├── CLOUDFLARE-WORKER.md      ← гайд: создать воркер и маршрут
├── TRACKING.md              ← гайд: настройка трекинга (таблица + переменные)
├── landing-structure.md      ← структура и тексты лендинга (для правок)
└── README.md
```
По желанию: создай `assets/` и положи туда `logo.svg` и `screenshot.png` (см. ниже).

## Что настроить перед запуском
**Значения уже подставлены** в `index.html` (1xCasino, 100 FS, 26MB, Android 7+, 57 959 загрузок, промокод `1xcTestCode`).
Осталось вписать только **`{{RATING}}`** — реальный рейтинг (например `4.8`). Промокод меняется в блоке B9. Таблица токенов — для справки:

| Токен | Что вписать | Пример |
|---|---|---|
| `{{APP_NAME}}` | Название приложения/бренда | `1WIN` |
| `{{BONUS}}` | Текст приветственного бонуса (только реальный оффер!) | `500% до 100 000 ₦` |
| `{{APK_SIZE}}` | Размер APK | `25 МБ` |
| `{{ANDROID_MIN}}` | Минимальная версия Android | `5.0` |
| `{{RATING}}` | Рейтинг в карточке приложения (только реальный!) | `4.8` |
| `{{DOWNLOADS}}` | Число загрузок в карточке (только реальное!) | `1 млн+` |

- **Кнопки CTA** ведут на `/go?b=<button>` (header, hero, install, bonus, final, sticky). Cloudflare Worker логирует клик и редиректит на реф-ссылку. Bemob больше не используется. Настройка — в `TRACKING.md`.
- **Логотип** подставлен по ссылке с CDN. Для надёжности на слабом интернете лучше скачать его в `assets/logo.svg` и заменить путь в трёх `<img>` на `assets/logo.svg`.
- **Скриншот** приложения: вставь свой `<img src="assets/screenshot.png" ...>` в блок `.phone__screen` вместо плейсхолдера.
- **Способы оплаты / ссылки Условия и Конфиденциальность** в футере — замени плейсхолдеры на реальные.

## Цвета и тема (стиль 1xBet/1xCasino)
Палитра реконструирована по фирменной системе бренда (**синяя база + ярко-зелёная кнопка загрузки + золотые акценты для бонуса**), т.к. сам референс закрыт от автоматического доступа — точные hex из его вёрстки вытащить не удалось.
Все цвета вынесены в один блок `:root` вверху `<style>` в `index.html`. Если есть точные бренд-оттенки — поменяй их там, и вся страница перекрасится автоматически:

| Переменная | Назначение |
|---|---|
| `--blue` / `--blue-2` | фирменный синий: иконки, ссылки, синие кнопки, акценты |
| `--green` / зелёный градиент `.btn` | основная кнопка «Скачать» |
| `--gold` / `--gold-2` | бонус и звёзды рейтинга |
| `--bg` / `--bg-alt` / `--surface` | фон и поверхности |

Стиль кнопок (скруглённый прямоугольник, верхний/uppercase-текст) задан в `.btn` — там же радиус (`--btn-radius`) и тени.

## Деплой на GitHub Pages
1. Создай репозиторий (например `landing`) и залей в него содержимое этой папки.
2. `Settings → Pages → Build and deployment → Source: Deploy from a branch`.
3. Branch: `main`, папка `/ (root)` → Save.
4. Через минуту страница доступна по адресу `https://<username>.github.io/<repo>/`.

Альтернативы (тоже бесплатно и быстро): Netlify / Cloudflare Pages — просто перетащи папку.

## Cloudflare Worker: гео-заголовок + трекинг
Один воркер (`worker.js`) делает три вещи на edge, без JS на странице и без задержек:
- **Гео в заголовке** — подставляет страну/город в слоты `<span data-geo="country">` / `<span data-geo="city">` (есть дефолты на случай пустого гео). Названия мест на английском.
- **Трекинг** — заходы (все + уникальные), клики по кнопкам, гео → пишет в Google-таблицу.
- **Редирект `/go`** — клик по кнопке логируется и редиректит на реф-ссылку.

Гайды: создать воркер и маршрут — **`CLOUDFLARE-WORKER.md`**; настроить трекинг (таблица, Apps Script, переменные) — **`TRACKING.md`**.

## Проверка перед запуском (быстрый QA)
- [ ] Все 6 кнопок ведут на `/go?b=...`, клик логируется и идёт редирект на реф-ссылку.
- [ ] Токены `{{...}}` заменены — на странице не осталось двойных фигурных скобок.
- [ ] Логотип и скриншот загружаются.
- [ ] Открыть на реальном Galaxy A22 (Chrome) — нет горизонтального скролла, кнопки крупные, текст читается.
- [ ] Проверить вес и скорость: PageSpeed Insights / Lighthouse, режим Mobile, эмуляция «Slow 4G».
- [ ] Кнопки удобно нажимать большим пальцем (тап-зоны ≥ 48px — уже заложено).
- [ ] Липкая кнопка «Скачать» видна на мобильном и скрыта на десктопе.

## Важно (compliance)
- Возрастное ограничение **18+** и строка про ответственную игру уже в футере — не удаляй их.
- Тексты честные, без обещаний выигрыша и фейковых «победителей в прямом эфире». Не добавляй ложные цифры выигрышей и фейковые таймеры — это вредит доверию и нарушает правила большинства рекламных сетей.
- Реклама азартных игр ограничена в Google Ads / Meta и запрещена в части стран — проверь правила сети и легальность по каждому гео до запуска трафика.
- Прямая установка APK требует от пользователя разрешения «неизвестные источники» — поэтому в лендинге есть блок-инструкция; это снижает отвал на установке.
